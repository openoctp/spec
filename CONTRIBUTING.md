# Contributing to the OCTP Specification

Thank you for helping improve the Open Contribution Trust Protocol.

## What we welcome

- Clarifications to ambiguous spec language
- Bug reports where the spec contradicts itself
- Examples that improve understanding
- RFC proposals for significant changes

## What requires an RFC

Any change that does one of these requires an RFC:
- Adds or removes a required field from the envelope schema
- Changes the signing model or integrity section
- Modifies allowed enum values for provenance method or review level
- Changes how verification fields are generated

Everything else can be a normal pull request.

## How to open an issue

Search existing issues before opening a new one.

For spec questions use the spec question issue template.
For RFC proposals use the RFC issue template.

## How to open a pull request

- Reference the issue your PR addresses
- Keep changes small and focused — one thing per PR
- Explain your reasoning, not just what you changed
- Update examples if your change affects the envelope schema

## Review process

You will receive an initial response within 72 hours.
RFC proposals remain open for a minimum of 14 days before any decision.
